# v0.4.0 Release Notes

**Release Date:** 2026-05-25

## Overview

MultiSpec v0.4.0 introduces Context Sources for grounding spec synthesis in actual codebase data, a complete Reconciliation Engine with conflict detection, and target adapters for GSD, GasTown, and GasCity. This release also adds MkDocs documentation generation with graph metrics and profile export capabilities.

## Highlights

- Context Sources for grounding spec synthesis in actual codebase data
- Reconciliation Engine with conflict detection and resolution
- Target Adapters: GSD, GasTown, GasCity, SpecKit constitution sync
- MkDocs documentation generation with graph metrics

## Features

### Context Sources

Ground technical spec synthesis in actual codebase data:

```yaml
# multispec.yaml
context:
  repositories:
    - path: "."
      include_structure: true
      include_deps: true
      include_apis: true

  graphize:
    enabled: true
    dir: ".graphize"

  files:
    - path: "docs/architecture.md"
      type: "architecture"
    - path: "docs/adr/*.md"
      type: "adr"

  mcp_servers:
    - name: "jira"
      command: "npx"
      args: ["-y", "@anthropic/mcp-jira"]
      tools: ["get_issue"]
```

**Context CLI Commands:**

```bash
# Gather context from configured sources
multispec context gather

# Show context summary
multispec context show

# Save snapshot for CI reproducibility
multispec context save

# List configured sources
multispec context sources
```

### Reconciliation with Conflict Detection

The reconciliation engine now detects conflicts across specs:

- Performance vs feature conflicts (PRD/TRD)
- Security vs usability conflicts (UXD/TRD)
- Scope vs timeline conflicts (MRD/PRD)
- Missing traceability (requirements not covered in TRD)

```bash
# Run reconciliation with conflict detection
multispec reconcile -p myproject

# Output includes spec.eval.json with:
# - Detected conflicts
# - Resolution decisions
# - Decision log
```

### Target Adapters

Export to multiple execution systems:

| Target | Output | Use Case |
|--------|--------|----------|
| `speckit` | spec.md, plan.md, tasks.md | GitHub Spec-Kit workflows |
| `gsd` | PLAN.md, STATE.md, config.json | Get Shit Done methodology |
| `gastown` | formula.toml, beads | Multi-agent formulas |
| `gascity` | city.toml | Multi-agent city orchestration |

```bash
# Export to GSD format
multispec export gsd -p myproject

# Export to GasTown
multispec export gastown -p myproject

# Export with constitution sync (SpecKit)
multispec export speckit -p myproject
```

### MkDocs Documentation Generation

Generate MkDocs-compatible documentation:

```bash
# Generate index.md for all projects
multispec docs generate

# Generate with graph metrics
multispec docs generate --with-graph

# Generate for specific project
multispec docs project myproject
```

### Spec Scaffolding

Create new specs from templates:

```bash
# Create a new MRD
multispec create mrd -p myproject

# Create a TRD (force overwrite)
multispec create trd -p myproject --force
```

### Profile Export

Extract profiles for customization:

```bash
# Export enterprise profile
multispec profiles export enterprise ./my-profile

# Use custom profile
multispec init my-project --profile-dir ./my-profile
```

### CONSTITUTION.md Multi-Location

Constitution files are discovered automatically:

1. Repo-level: `docs/specs/CONSTITUTION.md`
2. Org-level: `~/.config/multispec/CONSTITUTION.md`

### GraphMetrics in Status Reports

Status reports now include traceability metrics:

```bash
multispec status -p myproject

# Shows:
# - Total requirements, user stories, constraints
# - Trace coverage percentage
# - Conflict count
```

### Custom Rubrics Configuration

Configure rubrics per-project in `multispec.yaml`:

```yaml
rubrics:
  directory: "./rubrics"
  overrides:
    prd: "./custom-prd.rubric.yaml"
  strict_mode: true
  max_critical: 0
  max_high: 2
```

## Installation

```bash
go install github.com/plexusone/multispec/cmd/multispec@v0.4.0
```

## Migration Guide

No breaking changes. Existing projects continue to work.

To use new features:

1. **Context Sources**: Add `context` section to `multispec.yaml`
2. **Custom Rubrics**: Add `rubrics` section to `multispec.yaml`
3. **Profile Export**: Use `multispec profiles export` to extract and customize
